# Release v2.23.0 — Factory Director Reliability

**Date:** 2026-06-26
**Headline:** Kills the class of false/looping factory-director notices — false completions, ~30s re-fire spam, false-idle on live workers, supervisor-mislabeled-as-worker, and zombie shutdowns. EPIC `cas-fb94`.

## Why this matters
A single held task could generate 30–96 identical director nudges (observed live across sessions), flooding supervisors and workers and contributing to a wrong task-reset. This release makes director notices edge-triggered, role-aware, and reconciled with live state.

## Fixes

### Director notice correctness (EPIC cas-fb94)
- **TaskCompleted edge-trigger + state-guard** (`cas-55dc`) — `TaskCompleted`/`TaskAssigned` are now announced once per genuine transition via per-session guard sets that are never cleared on active-set oscillation. Ends the ~30s re-fire loop.
- **WorkerIdle live-worker gate** (`cas-4038`) — a worker that is task-less *between turns* but has a fresh heartbeat AND recent activity is no longer flagged idle; only a sustained cold window fires `WorkerIdle`.
- **Supervisor never nudged as an idle worker** (`cas-b67d`, `cas-405f`) — idle/assign nudges are role-gated to genuine workers; the zero-ready nudge no longer advises "close the epic" (stale-snapshot risk) and redirects to a live `task action=ready`. The config-only `director` member no longer renders as a live peer (`backend_type=None`) and its envelope color matches config (white, not green).
- **Shutdown kills the process tree** (`cas-8c5a`) — `shutdown_workers` now `killpg`s the worker's process group (honoring `force`), so a "shut down" Codex worker's `node→codex` tree is actually gone instead of lingering as a token-burning zombie.

### Factory observability + safety (cas-86c5)
- `worker_status` now shows a per-worker **"last activity: Xs ago (phase)"** line so an actively-investigating worker (clean diff, fresh checkpoints) is distinguishable from a stall.
- Per-turn git-state snapshots are labeled **"checkpoint"** instead of the misleading **"session-stop"**.
- **`task action=reset` alive-worker guard** — refuses to reset a task whose assignee has a fresh heartbeat unless `force=true` (new param), preventing orphaning of in-flight work. Dead-session recovery is unaffected.

### Developer experience
- **Commit guard denylist** (`cas-7e7b`) — factory workers can now commit on `feature/*`, `epic/*`, `fix/*`, and any named branch; only `main`/`master`/`staging`/detached HEAD are blocked. Denial messages explain the `--no-verify` limitation and which branch to switch to.

### Bonus test/hardening fixes
- Supervisor builtin guidance trimmed back under the 8 KB SessionStart cap (`cas-a7fa`).
- `skill_store` hash-id UNIQUE-collision flake fixed with a per-process sequence counter (`cas-6c0a`).

## Known issues
- `spawn_prep_cwd_is_worktree_for_four_workers` remains flaky under full-suite concurrency (process-global cwd race, test-only; tracked in `cas-7062`). Passes in isolation; no runtime impact.

## Operator note
The running factory must be rebuilt and `cas serve` restarted to pick up the new director binary — the fixes do not take effect in an already-running session.
