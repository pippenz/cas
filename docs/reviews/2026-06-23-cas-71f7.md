### cas-fc52 — web-close reconcile on pull
- **P1 (data loss) — FIXED.** `reconcile_web_close` did `store.update(&remote_task)`, a full row overwrite that would clobber locally-authored, not-yet-pushed `notes`/`description`/`acceptance_criteria` when a teammate's web close arrived. Fix: merge ONLY the close signal (`status`, `close_reason`, `closed_at`, `assignee`) onto the local row; the web close is authoritative about the close, not the body. Regression test `reconcile_preserves_locally_authored_content`.
- **P2 (PSR bypass) — addressed.** A web close on a `PendingSupervisorReview` task force-closes it past the review gate. Kept the behavior (an explicit teammate close is a human override) but now emits a `tracing::warn!` so the bypass is auditable; the merge fix also means the `review_envelope` in deliverables is preserved.
- Wire-shape check (highest value): `closed_via` DOES arrive at the JSON top level (server flattens `sync_entities.data`; same level `entity_matches_project` and `Task` deserialize use). Feature fires. ✓
- No loop/double-close (own pushes never carry `closed_via`). Idempotent on already-closed. Lease left to GC is inert (closed tasks aren't claimable).

### cas-7d54 — read-only comment mirror
- **P1 (silent comment loss) — FIXED.** `size: u64` failed deserialization on a float (`1234.0`, which a Next.js/Postgres JSONB stack can emit) or a numeric string, cascading to drop ALL comments on the task. Fix: `de_lenient_u64` tolerant deserializer (int/float/string/null → best-effort u64, negatives clamped to 0). Regression test `lenient_size_tolerates_float_and_string`.
- **P1-A (hot-path latency) — partially addressed.** Network fetch on every `task show`; timeout tightened 4s→3s to bound the logged-in-but-offline case. Early guards already short-circuit before any network I/O when not logged in / no team. Full fix (config-cached enablement check to skip the per-show file reads) noted as a follow-up.
- `spawn_blocking` + `.await` + `unwrap_or_default()` is correct (panics caught by `JoinError`, MCP server preserved). Team UUID in the URL path is correct. Read-only, graceful degradation confirmed.
