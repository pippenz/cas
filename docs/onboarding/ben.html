Connect to the box

Your user is ben on a Hetzner server at 87.99.156.244 (nickname: starscream). Your SSH public key is already in authorized_keys, so from your laptop: 

ssh ben@87.99.156.244

On your Mac, any terminal will do — Terminal.app, iTerm2, or Ghostty — SSH is built in. Once you're connected, you land in zsh with oh-my-zsh. From here on out, every command in this guide runs inside that SSH session, not on your laptop.

Tip
The box is also reachable as starscream. Add this to ~/.ssh/config on your laptop and you can just type ssh ben@starscream: 
Host starscream
  HostName 87.99.156.244
  ServerAliveInterval 60
(No User line — that way ssh ben@starscream and ssh daniel@starscream both work.)
2

Log into Claude Code

claude is already installed at /usr/bin/claude (version 2.1.100). You need to log in with your own Claude account — ours is currently banned, which is exactly why the Slack bridge is off.

Start Claude once. It'll print a login URL — open it on your phone or laptop, approve, and the CLI picks up the session. 

claude

When it finishes, quit with Ctrl+C twice, or type /exit. You won't usually talk to Claude directly — CAS will spawn it for you. We just need the credentials on disk.

Heads up
If you don't already have a Claude Code subscription, sign up at claude.ai first and pick a plan that includes Claude Code (Max or Pro). Free tier won't cut it.
3

GitHub (already authed)

A fine-grained PAT for your account is already loaded into every shell via ~/.zsh_secrets — exported as GH_TOKEN and GITHUB_TOKEN. gh picks it up automatically, and git uses it for HTTPS clones/pushes. You can confirm with: 

gh auth status
gh api user --jq .login   # should print: bhriv

Because HTTPS auth is covered, prefer the HTTPS clone URL (see step 5). If you ever want SSH-based git instead, your keypair is at ~/.ssh/id_ed25519 and you can register it with: 

gh ssh-key add ~/.ssh/id_ed25519.pub --title "starscream-ben"
Don't run
gh auth login — it'll fight the env-var auth. If GH_TOKEN is set (and it is, in every new shell), gh is already logged in.
4

Set your git identity

Commits need a name and email. Use whatever you use on your laptop so the attribution lines up: 

git config --global user.name  "Ben Your-Last-Name"
git config --global user.email "you@example.com"
git config --global init.defaultBranch main
git config --global pull.rebase false
5

Clone a project

Your workspace is ~/projects. Pick whatever you're working on and clone it there. For example: 

cd ~/projects
git clone https://github.com/pippenz/cas-src.git
cd cas-src

If you don't have access to a given repo yet, ping me and I'll add you as a collaborator. Then git clone will just work.

Note
CAS keys off the directory you're in, so the repo you want to work on should be the one you cd into before launching CAS.
6

Launch CAS

From inside a project directory, just run: 

cas

That opens the TUI. You get a task panel on the left, a pane grid on the right, and a command bar at the bottom. Quick orientation:

  • New Claude session — press n to spawn a pane running claude, scoped to this project.
  • Switch panes — Ctrl+←/→ cycles focus.
  • Scroll a pane — mouse wheel, or PgUp/PgDn.
  • Select text — hold Shift and drag to copy with your terminal's native selection.
  • Command palette — : then type a command (try :help).
  • Quit — Ctrl+Q.

Your tasks, memories, and search history live in ~/.cas/. First launch creates the DB. Everything persists across SSH sessions.

7

Keep sessions alive

If your laptop sleeps or your wifi drops, a raw SSH session dies and takes CAS with it. Wrap it in tmux so you can reattach: 

# Start or reattach a named session
tmux new -A -s work

# Inside tmux, launch CAS as normal
cd ~/projects/cas-src && cas

Detach with Ctrl+b, then d. Next time you SSH in, tmux attach -t work drops you right back where you left off — CAS still running, panes still alive.

8

When something breaks

A few things to try before pinging me:

  • CAS won't start or acts weird — cas --version to confirm the binary is there, then cas doctor for a self-check.
  • Claude sessions fail to authenticate — re-run claude standalone, log in again, then retry from CAS.
  • Can't clone a repo — ssh -T git@github.com to confirm your SSH key works; if that's fine, you probably just need collaborator access.
  • TUI rendering looks off — make sure your terminal is set to a reasonable font and has true-color enabled (iTerm2, Ghostty, WezTerm, Kitty all work well; Terminal.app is fine too).

If none of that works, grab the last few lines of output and send them over. I can SSH in as root and poke around from the other side.
